FD ARTIFACT CONTRACT (SCHEMA FD-ARTIFACT-1.0)

Gemini response MUST describe files via a JSON object with schema_version "FD-ARTIFACT-1.0".

Top-level fields:
- schema_version: "FD-ARTIFACT-1.0"
- work_item_id: "WI-###"
- producer_role: string
- artifact_type: "repo_patch" or "pipeline_snapshot"
- files: array of file entries
- delete: array of paths (may be empty)
- entry_point: optional string (for runnable app)
- build_command: optional string
- test_command: optional string
- verification_steps: array of ASCII strings
- notes: MUST be "" (empty string)

File entry:
- path: repo-relative path
- content: string (text or base64)
- content_type: "text/plain" | "application/json" | "text/html" | "text/css" | "text/javascript" | "text/markdown" | "application/xml" | "image/png" | "image/jpeg" | "application/zip" | "application/octet-stream"
- encoding: "utf-8" for text, "base64" for binary

Rules:
- For text files: encoding MUST be "utf-8" and content MUST be ASCII only.
- For binary: encoding MUST be "base64".
- No file omissions: every produced file MUST be declared in files[].
- All files MUST be fully specified (no placeholders like "three dots")..
