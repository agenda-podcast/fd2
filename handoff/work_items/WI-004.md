# Work Item: Schedule and Orchestrate News Summary

## Description

Configure the application to run automatically every 15 minutes, orchestrating the fetching, deduplication, summarization, and posting processes.

## Producer Role

DevOps Engineer

## Receiver Role

Product Manager

## Acceptance Criteria

- **Given** the application components are developed
- **When** the scheduled workflow is active
- **Then** the entire news summary process (fetch, deduplicate, summarize, post) runs automatically every 15 minutes.
- **And** the process adheres to the defined constraints (e.g., using GitHub Actions).

## Verification

- Verify the GitHub Actions workflow is configured and active.
- Monitor the execution logs of the scheduled workflow to confirm it runs every 15 minutes.
- Check that news summaries are posted to the Telegram channel at the expected intervals.

## Evidence

- GitHub Actions workflow configuration file (e.g., `.github/workflows/news-summary.yml`).
- Execution logs from GitHub Actions showing successful scheduled runs.
- Telegram channel messages confirming timely delivery of news summaries.
